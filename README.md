## What is this?

A fully systematic, rules-based FX trading strategy built from first principles.

Two independent signals must fire simultaneously before any trade is taken:

1. **Support/Resistance Zone** — price is at a historically significant swing level
2. **MACD Divergence** — momentum is moving opposite to price (exhaustion signal)

When both agree → market order entry → hold until Take Profit (2×ATR) or Stop Loss (1×ATR).

Validated using **Walk-Forward Optimization** (6 folds) — not simple backtesting — to prove out-of-sample performance and guard against curve-fitting.

## Key Design Decisions

### Why two signals instead of one?

A single signal can fire randomly. Two **completely independent** signals measuring different things (price memory vs momentum exhaustion) agreeing simultaneously is statistically significant. One witness can be wrong. Two independent witnesses describing the same event is much stronger evidence.

### Why Walk-Forward Optimization instead of backtesting?

Standard backtesting optimises parameters on the same data it tests on — this is curve-fitting (memorising exam answers). WFO trains on historical data, then tests on future data the optimizer **never saw**. The OOS results are the honest performance. This is the industry-standard approach used by professional quant funds.

### Why ATR-based SL and TP?

ATR (Average True Range) measures current daily volatility. Using ATR for exit distances means:
- **High volatility market** → wider SL/TP → trade has room to breathe
- **Low volatility market** → tighter SL/TP → capital isn't tied up unnecessarily

The same logic works across all pairs and all market conditions without manual adjustment.

### What is MACD Divergence?

When price makes a new **lower low** but MACD makes a **higher low** at the same time — sellers pushed price down with less momentum than before. They're running out of steam. This exhaustion of selling pressure (bullish divergence) often precedes a reversal. We use it as our momentum signal.

It fires on approximately **2% of all bars** — rare enough to be meaningful, not so rare that the system never trades.

### Bar-by-bar simulation (not vectorised)

The backtest engine processes one bar at a time, in strict chronological order, with no access to future data. This is the only way to correctly simulate a limit-order fill condition and avoid lookahead bias. Vectorised backtests are faster but statistically dishonest.

## Usage

```bash
# Run backtest on all pairs (default parameters)
python main.py --mode backtest

# Run backtest on a single pair
python main.py --mode backtest --pair EURUSD=X

# Run Walk-Forward Optimization (takes 5–15 minutes)
python main.py --mode wfo

# Check live signals right now
python main.py --mode live

# Force re-download data (ignore cache)
python main.py --mode backtest --refresh
```

## Integrity Check

The TP/SL ratio check runs automatically on every backtest:
```
TP distance / SL distance must equal 1.95–2.05
```
If outside this range → the code stops and logs `BUG!`. This catches any
reference-price error before it contaminates results.
